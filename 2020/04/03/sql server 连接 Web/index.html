SQL SERVER 连接 ASP.NET

开发应用时的两种方式

dataset概念：内存数据库，是表的查询结果的视图集 合，一个dataset是存在于内存中的，用于临时存放用户查询的结果，第三方语言从它里面再提出数据到用户界面。一个dataset可以存放若干个视图。
磁盘上的数据库叫物理数据库，是提供数据最原始的来源。内存数据库dataset是内存临时的数据库。一但从物理数据库把数据载入了dataset，可以与物理数据库断开连接，数据库可以被其它用户或APP使用，这种方法远比datareader高效，共享性好。
Datareader 是一种不断开数据库的应用，在整个数据读取过程中，它是保持与物理数据库连接状态，所以数据库独占，共享性差。
而dataset可以快速地把物理数据库载入，然后物理数据库不需要保持与dataset 的连接状态。每个用户可以在服务器上独立地建立一个自己的内存数据库。
打个比方：多个人在一个盆子里吃饭。datareader
多个人从盆子里把饭载入自己的小碗里，然后再慢慢吃.dataset
